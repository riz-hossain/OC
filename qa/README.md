## Priorities

Every AI-authored feature change must create or update manual and automated test evidence by default. Start with unit, API contract, parser/business logic, and integration tests. Add Playwright web E2E when a critical UI workflow changes. Add ZeuZ-compatible candidates for cross-surface Web/API/DB/Mobile/Desktop flows. Use `automation/verify_zkme_qa_contract.py` locally or in CI to make those rules fail-fast instead of advisory.

ZKME also generates deterministic lead-QA artifacts without using AI. Treat `feature-architecture-map.json`, `workflow-map.json`, `test-oracles.json`, `risk-matrix.json`, `test-data-matrix.json`, `traceability-matrix.json`, `test-design-quality.json`, and `manual/test-cases/lead-qa-generated/` as the minimum test-design floor. AI agents may improve these cases, but they must not replace them with shallow happy-path-only coverage.
